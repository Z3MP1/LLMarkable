# Overview  
The Document Conversion Pipeline is a Python-based tool that transforms source files (PDF, HTML, DOCX, PPTX, Images) into high-quality, LLM-friendly Markdown chunks with optional AI-powered content synthesis. It addresses the common problem of poorly structured document conversion that leads to information loss and suboptimal chunking for LLM consumption. This tool is designed for developers and data scientists who need to prepare documents for retrieval-augmented generation (RAG) systems and AI-powered content workflows.

# Core Features  
## Comprehensive Format Support ✅ COMPLETE
- **5 Pipeline Types**: PDF, HTML, DOCX, PPTX, Image processing with OCR
- **12 File Extensions**: .pdf, .html, .htm, .docx, .pptx, .png, .jpg, .jpeg, .tiff, .tif, .bmp, .gif
- **Format-Specific Optimization**: Each pipeline leverages format-specific features and capabilities
- **Extensible Architecture**: Easy addition of new format handlers via plugin system

## Intelligent Chunk Management ✅ COMPLETE
- **Token-based chunking**: Uses precise token counting (not character-based) for optimal chunk sizing
- **Smart consolidation**: Merges small chunks with neighbors to prevent information loss
- **Research-backed parameters**: chunk_size (2048 tokens), min_tokens (330), chunk_overlap (100)
- **Quality filtering**: Intelligent filtering based on content density and token requirements

## AI-Augmented Content Synthesis 🎯 PHASE 2 IN PROGRESS
- **LLM Integration**: Local and cloud LLM support for content enhancement (Ollama, OpenAI, Anthropic, Gemini)
- **Content Refinement**: Transform raw chunks into coherent, readable documents
- **Quality Preservation**: Maintain factual accuracy while improving readability
- **Configurable Enhancement**: Multiple refinement levels and preservation modes
- **TokenManager**: Provider-agnostic token counting, batching, caching, and cost estimation (OpenAI, Ollama, etc.)
- **Comprehensive LLM Integration Tests**: End-to-end tests with mock LLM providers, covering chunk refinement, metadata injection, error handling, and performance
- **Synthesis Engine Integration**: All pipelines now support optional LLM-powered synthesis with format-specific prompt templates and metadata injection
- **OpenAIProvider**: Integration in progress (tiktoken-based token counting, robust error handling, async support)

- Graceful fallback mechanisms (provider, model, cached response)

- Provider-specific token counting strategies (tiktoken for OpenAI, approximation for Anthropic)
- Request batching and response caching
- Context window optimization and efficient prompt engineering

# Abstract Provider Pattern (Industry Standard)
from abc import ABC, abstractmethod

class BaseLLMProvider(ABC):
    @abstractmethod
    async def generate(self, prompt: str, **kwargs) -> str:
        pass
    
    @abstractmethod
    def get_token_count(self, text: str) -> int:
        pass

# LangChain LCEL Pattern for Chainable Operations
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

class ContentSynthesizer:
    def __init__(self, provider: BaseLLMProvider):
        self.provider = provider
        # Chainable components using LCEL
        self.chain = (
            ChatPromptTemplate.from_template("Enhance this content: {content}")
            | provider
            | StrOutputParser()
        )
